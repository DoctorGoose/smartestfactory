When starting a new game using the Main Menu you are presented with a screen where you can select one of four different starting locations, along with a session name and session type, plus optionally choose to skip onboarding, set multiplayer settings, and/or enable advanced game settings.

Starting location

The player can choose one of four locations to start in. They are all located on the same Map , and regardless of the choice, the player will be able to access any of these in the course of the game.

Start Area Name Image Best For Description Notes / Remarks
1 Grass Fields Ideal for first-time pioneers The first viable building location FICSIT had found, standing out because of its open and relatively flat fields. Considering the distances between mining locations, building outposts and/or large transportation lines may be necessary eventually. [ note 1 ]
2 Rocky Desert Suitable for first-time pioneers The textbook example of a FICSIT-approved building location, with balanced resources, space, and distances. Additionally, it is very close to the sea, making water an easy commodity. [ note 2 ]

Session name

Session names need to be at least 3 characters long and are contain within the Save File Header .

Session names are displayed in autosave file names in the format of {SESSION NAME}_autosave_#.sav where "#" is a value of 0, 1, or 2.

Manual game saves retain the session name internally, however the save file name can be anything desired by the player.

Once a session name is entered the Start Game button is highlighted in orange and be clicked to begin in the starting location selected.

Naming convention

  • Session names can use most characters. except as indicated below.
  • CAUTION To avoid game issues, session names should not start or end with a Space or a . (period). Additionally, session names should not contain an ' (apostrophe), or any of the following Windows Reserve Characters:
    • < (less than)
    • > (greater than)
    • : (colon)
    • " (double quote)
    • / (forward slash)
    • \ (backslash)
    • Bar (vertical bar or pipe "|")
    • ? (question mark)
    • * (asterisk)

Session type

There are two options that are similar to Multiplayer Privacy Settings :

  • Private - this is the default setting , and only allows clients join the hosted session using this game save if invited using in-game invites.
  • Friends Only - this allows anyone in the host's friend list to join the hosted session using this game save.

Skip intro

By checking the checkbox the game will skip onboarding, and begin in the starting location selected with the HUB fully built, and Tier 1 unlocked.

Multiplayer settings

During the start of a new game save you can set Multiplayer settings for that game session as follows:

Option Default Effect
Single Player Single Player You want to play the game alone. Nobody will be able to join you, no matter what.
IP Advanced method of hosting a multiplayer game. Does not require login to any service but it might require setting up port forwarding. Others will be able to join you using your IP address that you will have to share with them independently.
Steam Requires login to Steam. Other Steam players might be able to join you, depending on visibility preferences.
Epic Requires login to Epic Games. Players from Steam or Epic Games will be able to join your session.

See also Multiplayer Settings .

Advanced game settings

During the start of a new game save you can optionally enable Advanced Game Settings .

CAUTION: It is not possible to disable Advanced Game Settings on a particular save file once they have been enabled.

  • Version 1.0 new game screen showing Advanced Game Settings warning.

Mod savegame settings

If game mods are loaded you will see MOd Savegame Settings> shown on the right. This contains options specific to various Game Mods.
